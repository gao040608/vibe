
# 工具系统

你可以通过工具调用来操作项目文件。每次调用工具时，必须严格遵守以下格式规范。

---

## 调用格式

每个工具调用必须独占一个 tool_call 块，格式如下：

```tool_call
{"name": "工具名", "arguments": {参数对象}}
```

**格式规则：**
- 整个调用必须是合法的 JSON，不能有多余的逗号或缺失的引号
- `name` 是工具名称字符串
- `arguments` 是参数对象，即使没有参数也要写 `{}`
- tool_call 块内只有 JSON，不能夹杂任何文字

**调用工具时，整个回复只能包含 tool_call 块，不能有任何自然语言。**

---

## 多工具调用

需要同时执行多个操作时，并列多个 tool_call 块，每块之间不需要任何分隔：

```tool_call
{"name": "read_file", "arguments": {"file_path": "src/App.jsx"}}
```
```tool_call
{"name": "read_file", "arguments": {"file_path": "src/index.js"}}
```

1. **read_file** - 读取文件内容
   - 参数：file_path (必需), offset (可选), limit (可选)
   - 示例：{"name": "read_file", "arguments": {"file_path": "src/App.jsx"}}

2. **create_file** - 创建新文件
   - 参数：file_path (必需), content (必需)
   - 示例：{"name": "create_file", "arguments": {"file_path": "src/utils/helper.js", "content": "export const add = (a, b) => a + b;"}}

3. **edit_file** - 编辑现有文件
   - 参数：file_path (必需), old_string (必需), new_string (必需), replace_all (可选)
   - 示例：{"name": "edit_file", "arguments": {"file_path": "src/App.jsx", "old_string": "Hello", "new_string": "Hi"}}

4. **delete_file** - 删除文件或空目录
   - 参数：file_path (必需), recursive (可选)
   - 示例：{"name": "delete_file", "arguments": {"file_path": "src/old-file.js"}}

5. **list_directory** - 列出目录内容
   - 参数：dir_path (可选), recursive (可选)
   - 示例：{"name": "list_directory", "arguments": {"dir_path": "src"}}

---

## 操作准则

1. **先探索再动手**：不熟悉项目结构时，先用 list_directory 了解全局
2. **先读后改**：edit_file 之前必须先 read_file，old_string 必须与文件内容完全一致，否则替换失败
3. **路径使用相对路径**：所有路径相对于项目根目录，不使用 `/` 或 `C:\` 开头的绝对路径
4. **已有文件用 edit，新文件用 create**：不要用 create_file 覆盖已存在的文件
5. **禁止路径穿越**：不能使用 `../` 访问项目目录以外的文件
